\section{System Executive Summary}
DeepFake Defender is a layered detector for both images and videos.
It does not rely on one single model.

\subsection{Final Image Decision Order}
\begin{enumerate}
  \item C2PA signed metadata check,
  \item SynthID watermark check,
  \item AI model plus forensic calibration.
\end{enumerate}

Final policy currently implemented:
\begin{enumerate}
  \item If C2PA metadata is present, classify as AI-generated and skip lower layers.
  \item Else, if SynthID watermark is detected, classify as AI-generated and skip model layer.
  \item Else, use model plus forensics with conservative thresholding.
\end{enumerate}

\subsection{Final Video Decision Order}
\begin{enumerate}
  \item Try dedicated video checkpoints first.
  \item If unavailable, fallback to frame-sampling using the image predictor.
\end{enumerate}

\section{Runtime Architecture and Boot Sequence}
\subsection{Backend Startup Sequence}
When \fpath{backend/app.py} starts:
\begin{enumerate}
  \item it computes backend and project paths,
  \item configures upload folder and max request size,
  \item attempts to load \texttt{AIEnsemblePredictor()} once,
  \item attempts to load \texttt{SynthIDService} once,
  \item exposes routes for template pages and API endpoints.
\end{enumerate}

This startup strategy keeps inference per request faster because heavy model objects are loaded once, not for every upload.

\subsection{Health Endpoint Contract}
\fpath{/api/health} returns:
\begin{enumerate}
  \item Python executable path and version,
  \item model load state for AI predictor and SynthID detector,
  \item C2PA runtime availability, Reader API presence, and import error (if any).
\end{enumerate}

This directly solves environment mismatch debugging.

\section{Image Pipeline: Behind-the-Scenes Full Flow}
This section describes exactly what happens after image upload.

\subsection{Request Entry and Safety Checks}
Endpoint: \fpath{/api/analyze} (POST).

The backend validates:
\begin{enumerate}
  \item file key exists in multipart request,
  \item filename is not empty,
  \item extension is allowed (png, jpg, jpeg, webp),
  \item payload size remains under 16MB.
\end{enumerate}

If checks pass, the file is saved using a UUID-based temporary name in \fpath{backend/uploads}.

\subsection{Result Object Initialization}
Before layer execution, backend builds a fixed response shape:
\begin{enumerate}
  \item \texttt{layers.c2pa}, \texttt{layers.synthid}, \texttt{layers.ai_model},
  \item \texttt{final_verdict}, \texttt{confidence}, \texttt{is_ai_generated},
  \item optional \texttt{explainability} and later \texttt{forensic_summary}.
\end{enumerate}

This stable contract helps both template and React frontends render consistent UI.

\subsection{Layer 1: C2PA}
The backend calls \texttt{check_c2pa(filepath)}.

Status logic:
\begin{enumerate}
  \item if C2PA package unavailable: status becomes \texttt{unavailable},
  \item if manifest present: status \texttt{verified} when valid, else \texttt{present},
  \item if no manifest: status \texttt{not_found}.
\end{enumerate}

Decision gate:
\begin{enumerate}
  \item if \texttt{c2pa_present = true}, backend sets confidence to 100,
  \item sets \texttt{is_ai_generated = true},
  \item sets verdict as either ``AI Generated (C2PA AI Declaration)'' or ``AI Generated (C2PA Metadata Present)'',
  \item marks SynthID and AI model layers as skipped.
\end{enumerate}

So C2PA presence is decisive by policy.

\subsection{Layer 2: SynthID}
Only executed when C2PA metadata is not present.

Flow:
\begin{enumerate}
  \item call \texttt{synthid_detector.analyze(filepath)},
  \item if status complete and watermark detected: final verdict becomes ``AI Generated (SynthID Watermark Detected)'',
  \item AI model layer is skipped,
  \item if negative/unavailable/error: continue to model layer decision path.
\end{enumerate}

\subsection{Layer 3: AI Model + Forensics}
Only executed when neither C2PA nor SynthID has already produced an AI-positive decisive result.

Flow:
\begin{enumerate}
  \item call \texttt{predictor.predict(filepath, return_details=True)},
  \item include model scores, forensic module scores, and artifacts in response,
  \item compute final label from calibrated threshold logic,
  \item optionally apply pipeline override for weak AI-only evidence.
\end{enumerate}

\subsection{Important Low-Confidence Override Rule}
In \fpath{backend/app.py}, if all of these are true:
\begin{enumerate}
  \item model label is AI Image,
  \item model confidence is below 60,
  \item no forensic support modules,
  \item forensics-only score below 45,
  \item SynthID is complete and negative,
\end{enumerate}
then backend overrides to Real Image.

This guardrail is critical for reducing false positives.

\subsection{Forensic Summary and Cleanup}
After layer logic:
\begin{enumerate}
  \item backend generates \texttt{forensic_summary} with human-readable points,
  \item temporary upload file is removed in \texttt{finally},
  \item complete JSON result is returned to frontend.
\end{enumerate}

\section{Core Module Internals (Image Path)}
\subsection{C2PA Internals: \fpath{backend/src/c2pa_checker.py}}
Key technical behavior:
\begin{enumerate}
  \item import-time C2PA flags: \texttt{C2PA_AVAILABLE}, \texttt{C2PA_IMPORT_ERROR},
  \item runtime status helper \texttt{get_c2pa_runtime_status()},
  \item robust manifest parsing from dict or JSON string,
  \item AI assertion extraction scans \texttt{c2pa.actions} for \texttt{trainedalgorithmicmedia} digital source type.
\end{enumerate}

Return object includes:
\begin{enumerate}
  \item \texttt{c2pa_present}, \texttt{available}, \texttt{valid}, \texttt{issuer},
  \item \texttt{ai_generated} when explicit declaration exists,
  \item \texttt{raw_data} for deeper inspection.
\end{enumerate}

\subsection{SynthID Service Internals: \fpath{backend/src/synthid/service.py}}
Service responsibilities:
\begin{enumerate}
  \item verify codebook exists at startup,
  \item initialize robust extractor,
  \item normalize detector output fields and confidence to percentage,
  \item provide health metadata for debugging.
\end{enumerate}

Output includes advanced fields:
\begin{enumerate}
  \item \texttt{correlation}, \texttt{phase_match}, \texttt{structure_ratio},
  \item \texttt{carrier_strength}, \texttt{multi_scale_consistency}.
\end{enumerate}

\subsection{Robust SynthID Extractor: \fpath{backend/src/synthid/robust_extractor.py}}
This module is intentionally advanced and multi-stage.

Main components:
\begin{enumerate}
  \item multi-scale analysis (256, 512, 1024),
  \item multi-denoiser fusion (wavelet families, bilateral, non-local means, Wiener),
  \item carrier peak detection with scale voting,
  \item ICA/PCA watermark component separation,
  \item adaptive thresholding using learned codebook statistics.
\end{enumerate}

Core decision checks in \texttt{detect_array()}:
\begin{enumerate}
  \item correlation exceeds codebook threshold,
  \item average phase match exceeds 0.45,
  \item structure ratio remains in expected range (0.7 to 2.0).
\end{enumerate}

Confidence combines four normalized terms:
\[
\text{confidence} = 0.35\cdot s_{corr} + 0.35\cdot s_{phase} + 0.15\cdot s_{structure} + 0.15\cdot s_{consistency}
\]

\subsection{Image Ensemble Predictor: \fpath{backend/combine_model.py}}
\subsubsection{Loaded Components}
\begin{enumerate}
  \item ResNet50 binary head (required),
  \item ViT classifier from \texttt{dima806/ai_vs_real_image_detection} (optional fallback-safe),
  \item meta-learner + polynomial transformer (optional),
  \item custom forensic calibration module.
\end{enumerate}

\subsubsection{Score Fusion Logic}
If ViT exists:
\begin{enumerate}
  \item compute ResNet and ViT AI probabilities,
  \item produce base average,
  \item if meta-learner is available, use meta score with drift-aware stabilization,
  \item then calibrate with forensic output.
\end{enumerate}

If ViT is unavailable:
\begin{enumerate}
  \item system remains operational using ResNet plus forensic calibration.
\end{enumerate}

\subsubsection{Custom Forensic Weighted Score}
\fpath{backend/custom_forensics.py} applies:
\[
S_{weighted}=0.50S_{resnet}+0.20S_{frequency}+0.20S_{anatomy}+0.10S_{entropy}
\]

Then it applies calibration rules based on:
\begin{enumerate}
  \item supporting forensic modules,
  \item negative forensic modules,
  \item forensics-only AI probability.
\end{enumerate}

\subsubsection{Dynamic AI Threshold Resolution}
Threshold is not fixed.
It increases strongly when forensic support is weak or negative.
For example:
\begin{enumerate}
  \item with no forensic support and weak forensics-only score, threshold can rise to 0.88 or 0.90,
  \item with strong multi-module forensic support, threshold can be as low as 0.55.
\end{enumerate}

\subsubsection{Decision Confidence Formula}
Confidence is margin-based and clipped:
\[
C = 0.5 + 0.5\cdot\text{clip}(m,0,1)
\]
where margin $m$ is computed relative to threshold on the predicted side (AI or Real).

\subsection{Custom Forensic Modules: \fpath{backend/custom_forensics.py}}
Three forensic analyzers support model calibration.

\subsubsection{Frequency Analyzer}
Performs FFT power-spectrum checks for checkerboard-like upsampling traces.
Uses mirrored peak pairs, grid energy ratio, and peak prominence.

\subsubsection{Anatomy Analyzer}
Uses MediaPipe landmarks for body/hands.
Scores asymmetry, implausible angles, landmark geometry anomalies, and out-of-frame landmark behavior.
Supports classic and Tasks API modes depending on environment.

\subsubsection{Entropy Analyzer}
Computes local entropy over patch grids.
Looks for suspicious center-edge detail gaps and patch spread abnormalities while avoiding false alarms on naturally low-detail scenes.

\subsection{Forensic Summary Generator: \fpath{backend/forensic.py}}
This module translates raw layer outputs into presentation-friendly statements.

Key behaviors:
\begin{enumerate}
  \item determines method source (C2PA policy, SynthID, or model),
  \item produces short ``points'' list for UI,
  \item includes conclusion wording that matches final verdict,
  \item provides separate summary logic for image and video results.
\end{enumerate}

\section{Video Pipeline: Behind-the-Scenes Full Flow}
\subsection{Request Entry}
Endpoint: \fpath{/api/analyze_video} (POST).

Validation checks:
\begin{enumerate}
  \item file exists in request,
  \item filename is present,
  \item extension is in mp4/avi/mov.
\end{enumerate}

File is saved to temporary UUID path and removed after analysis.

\subsection{Primary Path: Dedicated Video Checkpoints}
In \fpath{backend/video_detect_standalone.py}, function \texttt{deepfakes_video_predict()} first tries \texttt{_predict_with_checkpoint_model()}.

Requirements:
\begin{enumerate}
  \item \fpath{backend/checkpoints/efficientnet.onnx},
  \item \fpath{backend/checkpoints/model.pth},
  \item optional dependencies \texttt{onnx} and \texttt{onnx2pytorch}.
\end{enumerate}

Method:
\begin{enumerate}
  \item sample 3 frames,
  \item preprocess each frame to 256x256,
  \item run converted checkpoint model,
  \item average fake probability,
  \item classify FAKE when probability \(\ge 0.60\).
\end{enumerate}

\subsection{Fallback Path: Frame-Based Estimation}
If dedicated path fails, system automatically uses \texttt{_predict_with_frame_fallback()}.

Method:
\begin{enumerate}
  \item sample up to 6 frames across full clip,
  \item run image predictor probability for each frame,
  \item count flagged frames using frame threshold 0.68,
  \item combine mean and peak into final score:
\end{enumerate}

\[
P_{video} = 0.7\cdot P_{mean} + 0.3\cdot P_{peak}
\]

Video verdict threshold in fallback mode is 0.65.

\subsection{Video Explainability Output}
Result includes:
\begin{enumerate}
  \item frames analyzed,
  \item average AI probability,
  \item peak AI probability,
  \item flagged frame count,
  \item per-frame timestamps/probabilities (fallback path),
  \item forensic summary text from \fpath{backend/forensic.py}.
\end{enumerate}

\section{Common Questions with Ready Answers}
\subsection{Architecture Questions}
\textbf{Q1: Why use multi-layer detection instead of one model?}\\
Because provenance metadata and watermark evidence are often higher-confidence signals than pixel-model inference alone.

\textbf{Q2: Why is C2PA first?}\\
Because signed provenance can be decisive and avoids unnecessary lower-layer inference.

\textbf{Q3: Why can C2PA metadata presence alone return AI-generated?}\\
This is an explicit project policy change to make signed metadata decisive even without explicit AI action declaration.

\textbf{Q4: What prevents contradictory layer outputs?}\\
Short-circuit design in \fpath{backend/app.py}. When upper layers are decisive, lower layers are marked skipped.

\subsection{Model and Forensic Questions}
\textbf{Q5: How are false positives reduced?}\\
Conservative thresholding plus low-confidence override when SynthID is negative and forensic support is weak.

\textbf{Q6: Is ViT mandatory?}\\
No. If ViT fails to load, system still runs with ResNet and forensic calibration.

\textbf{Q7: What if meta-learner files are missing?}\\
System continues with base blend and forensic calibration; meta fusion is optional.

\textbf{Q8: What forensic checks are implemented?}\\
Frequency checkerboard artifacts, anatomical landmark plausibility, and entropy distribution consistency.

\subsection{Video Questions}
\textbf{Q9: How does video detection work when checkpoints are missing?}\\
It automatically falls back to frame sampling and image-model probability aggregation.

\textbf{Q10: How is final fallback video score computed?}\\
\(0.7 \times\) mean frame score plus \(0.3 \times\) peak frame score.

\textbf{Q11: Why return frame-level explainability?}\\
To avoid black-box output and show how clip-level verdict emerged from sampled frames.

\subsection{Operations Questions}
\textbf{Q12: Why add /api/health?}\\
To quickly inspect interpreter path, model load status, and C2PA availability during runtime troubleshooting.

\textbf{Q13: Why use Waitress for production mode?}\\
Flask dev server is not intended for production robustness.

\textbf{Q14: Why can cloud behavior differ from local?}\\
Free hosting cold starts, package/platform differences, and missing model artifacts can change runtime behavior.

\textbf{Q15: Why is virtual environment path emphasized?}\\
Starting backend with wrong interpreter is a common source of ``package installed but unavailable at runtime'' issues.

\section{Known Limits and Honest Expectations}
\begin{enumerate}
  \item free hosting cold starts cannot be fully eliminated,
  \item model and artifact availability must be guaranteed per environment,
  \item C2PA package support can vary by host runtime and platform,
  \item all detectors can still have edge cases; forensic calibration reduces but cannot mathematically eliminate risk.
\end{enumerate}

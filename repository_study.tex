\section{Runtime Architecture}
\subsection{High-Level Layered Pipeline}
Image analysis uses a strict short-circuit decision sequence:
\begin{enumerate}
  \item Layer 1: C2PA metadata check,
  \item Layer 2: SynthID watermark check,
  \item Layer 3: AI model plus forensic calibration.
\end{enumerate}

The first decisive AI-positive layer short-circuits lower layers.

Video analysis uses:
\begin{enumerate}
  \item dedicated checkpoint model path when checkpoints are present,
  \item fallback frame scoring path when dedicated checkpoints are missing or fail.
\end{enumerate}

\subsection{Asynchronous Initialization in \fpath{backend/app.py}}
At app startup:
\begin{enumerate}
  \item Flask server binds immediately,
  \item a background daemon thread starts \texttt{initialize\_all()},
  \item startup attempts Hugging Face downloads for missing assets,
  \item predictor and SynthID service are loaded,
  \item runtime ready flag is set when core objects are initialized.
\end{enumerate}

This design allows early health checks but may return temporary 503 for \texttt{/api/analyze} before model readiness.

\section{Image Analysis Runtime Flow}
\subsection{Request Validation}
\fpath{/api/analyze} accepts multipart file uploads with:
\begin{enumerate}
  \item extensions in \{png, jpg, jpeg, webp\},
  \item max request size 16 MB.
\end{enumerate}

If predictor is not ready:
\begin{enumerate}
  \item returns 503 while loading, or
  \item attempts runtime recovery and returns 503 if initialization still fails.
\end{enumerate}

\subsection{Layer 1: C2PA}
\texttt{check\_c2pa(filepath)} produces status and manifest-derived flags.

Decision policy:
\begin{enumerate}
  \item if \texttt{c2pa\_present = true}, image is finalized as AI generated,
  \item SynthID and model layers are marked skipped.
\end{enumerate}

Verdict text differentiates:
\begin{enumerate}
  \item explicit AI assertion case,
  \item metadata-presence policy case.
\end{enumerate}

\subsection{Layer 2: SynthID}
Executed only when C2PA metadata is not present.

If SynthID returns calibrated watermark positive:
\begin{enumerate}
  \item final verdict is AI generated,
  \item model layer is skipped.
\end{enumerate}

Otherwise pipeline continues to Layer 3.

\subsection{Layer 3: Model + Forensics}
\texttt{AIEnsemblePredictor.predict(..., return\_details=True)} provides:
\begin{enumerate}
  \item label (AI Image / Real Image),
  \item confidence and AI probability,
  \item threshold and score internals,
  \item forensic module outputs,
  \item explainability artifacts.
\end{enumerate}

\section{Model Fusion in \fpath{backend/combine_model.py}}
\subsection{Component Models}
\begin{enumerate}
  \item ResNet50 is required and loaded from \fpath{model\_output/resnet50\_finetuned\_benchmark.pth}.
  \item ViT is optional and loaded from Hugging Face model \texttt{dima806/ai\_vs\_real\_image\_detection} unless \texttt{SKIP\_VIT=1}.
  \item Meta learner is optional and requires both \fpath{ai\_detector\_meta\_learner.joblib} and \fpath{polynomial\_transformer.joblib}.
\end{enumerate}

\subsection{TTA Behavior}
If \texttt{ENABLE\_TTA=1}, the predictor evaluates:
\begin{enumerate}
  \item original image,
  \item mirrored image.
\end{enumerate}
Scores are averaged across views.

\subsection{Legacy and Meta Fusion}
Given ResNet score $p_r$ and ViT score $p_v$:
\[
p_{blend} = \frac{p_r + p_v}{2}
\]
(if ViT unavailable, $p_{blend}=p_r$).

If meta learner is available, $p_{meta}$ is computed from polynomial features of $(p_r, p_v)$.
Depending on version warning and drift, final legacy score uses direct or blended meta usage.

\subsection{Custom Forensic Blend}
Custom forensic module produces calibrated score $p_c$ and forensics-only score $p_f$.
If no supporting modules exist:
\[
p_{final,pre} = 0.70\,p_{legacy} + 0.30\,p_c
\]
Else stronger model-forensic behavior is used based on support level.

\subsection{Disagreement Guards}
Two guards refine score behavior:
\begin{enumerate}
  \item ViT disagreement guard: prevents false real outcomes in high-confidence AI disagreement contexts with forensic corroboration.
  \item Real disagreement guard: reduces false AI outcomes when forensic support is weak.
\end{enumerate}

Score-consensus refinement then adjusts selected disagreement cases.

\section{Dynamic Threshold Resolution}
\subsection{Base Threshold Bands}
Threshold is context-dependent (supporting modules, negative modules, forensics-only score, ViT availability).

Representative values with ViT available:
\begin{enumerate}
  \item strong support: 0.54 to 0.62,
  \item weak/no support: 0.68 to 0.76,
  \item very strong model scenarios may lower threshold when corroboration exists.
\end{enumerate}

\subsection{Low-Corroboration Safety Guard}
A specific guard raises threshold to 0.86 when all are true:
\begin{enumerate}
  \item no supporting forensic modules,
  \item anatomy module unavailable,
  \item at least one negative module,
  \item forensics-only score is weak,
  \item frequency and entropy support are weak.
\end{enumerate}

This avoids model-only false AI outcomes when corroborating forensic evidence is insufficient.

\subsection{Decision and Confidence}
Final decision:
\[
\text{AI if } p_{final} \ge t, \quad \text{else Real}
\]
where $t$ is the resolved threshold.

Confidence is computed from relative margin:
\[
\text{conf} = 0.5 + 0.5\,\mathrm{clip}(m,0,1)
\]
with
\[
m = \begin{cases}
\dfrac{p_{final}-t}{1-t}, & \text{AI decision} \\
\dfrac{t-p_{final}}{t}, & \text{Real decision}
\end{cases}
\]

\section{Video Analysis Runtime Flow}
\subsection{Dedicated Path}
In \fpath{backend/video_detect_standalone.py}, dedicated path requires both:
\begin{enumerate}
  \item \fpath{backend/checkpoints/efficientnet.onnx},
  \item \fpath{backend/checkpoints/model.pth}.
\end{enumerate}

It samples 3 frames and uses threshold 0.60.

\subsection{Fallback Path}
Fallback path samples 6 frames and scores each frame with the image predictor.

Let frame probabilities be $\{p_i\}$:
\[
p_{mean}=\frac{1}{N}\sum_i p_i, \qquad p_{peak}=\max_i p_i
\]
\[
p_{video}=\mathrm{clip}(0.7\,p_{mean}+0.3\,p_{peak}, 0, 1)
\]
Decision threshold for fallback is 0.65.

\section{Deep Technical Walkthroughs}
\subsection{End-to-End Image Workflow (Runtime Order)}
For every image request to \fpath{/api/analyze}, the control flow in \fpath{backend/app.py} is:
\begin{enumerate}
  \item validate multipart upload, extension, and request size,
  \item ensure predictor runtime is available (or return 503 while loading),
  \item store file in \fpath{backend/uploads/} with a UUID name,
  \item run Layer 1 C2PA check,
  \item if C2PA is present: finalize AI verdict and skip lower layers,
  \item otherwise run Layer 2 SynthID service,
  \item if SynthID is calibrated-positive: finalize AI verdict and skip model layer,
  \item otherwise run Layer 3 model fusion in \fpath{backend/combine_model.py},
  \item attach human-readable forensic summary from \fpath{backend/forensic.py},
  \item always delete temporary upload before response.
\end{enumerate}

\subsection{C2PA Deep Dive (\fpath{backend/src/c2pa_checker.py})}
\subsubsection{Runtime Capability Detection}
The module first attempts to import \texttt{c2pa}. It reports capability through:
\begin{enumerate}
  \item \texttt{available}: package import success,
  \item \texttt{has\_reader}: whether \texttt{c2pa.Reader} exists,
  \item \texttt{has\_c2pa\_error}: whether a library-specific exception class exists,
  \item \texttt{import\_error}: captured import failure text.
\end{enumerate}

\subsubsection{Manifest Parsing Procedure}
When available, \texttt{check\_c2pa(file\_path)} performs:
\begin{enumerate}
  \item open file with \texttt{Reader},
  \item parse manifest JSON/dict payload,
  \item locate \texttt{active\_manifest},
  \item evaluate \texttt{validation\_status} (empty list means valid),
  \item inspect \texttt{assertions} for \texttt{c2pa.actions},
  \item scan action \texttt{digitalSourceType} for token \texttt{trainedalgorithmicmedia}.
\end{enumerate}

The app-level policy is intentionally strict: any present C2PA manifest is treated as AI-generated evidence for layer short-circuiting.

\subsection{SynthID Deep Dive (Extractor + Service)}
SynthID logic is implemented in two layers:
\begin{enumerate}
  \item \fpath{backend/src/synthid/robust_extractor.py}: raw watermark evidence extraction,
  \item \fpath{backend/src/synthid/service.py}: calibrated decision policy and response shaping.
\end{enumerate}

\subsubsection{Raw Signal Extraction (\fpath{robust_extractor.py})}
For each input image, the extractor:
\begin{enumerate}
  \item resizes to codebook target size,
  \item extracts fused noise using wavelet, bilateral, non-local means, and Wiener denoisers,
  \item computes correlation to reference watermark noise,
  \item computes FFT magnitude/phase carrier evidence from extracted and known frequency carriers,
  \item computes structure ratio and multi-scale correlation consistency,
  \item applies gate-based watermark decision logic.
\end{enumerate}

Gate conditions are:
\[
m_{corr} = corr - t, \quad a = \max(0.0025,\;0.35\,\sigma_{corr})
\]
\[
\text{corr\_gate}:\; m_{corr} > 0.85a
\]
\[
\text{phase\_gate}:\; phase\_match > 0.50,
\quad
\text{carrier\_gate}:\; carrier\_match\_ratio > 0.28
\]
\[
\text{structure\_gate}:\; 0.75 < structure\_ratio < 1.95,
\quad
\text{consistency\_gate}:\; multi\_scale\_std < 0.19
\]

Raw watermark-positive requires all gates to pass.

Raw confidence is a weighted composition:
\[
conf = 0.28s_{corr} + 0.24s_{phase} + 0.18s_{carrier} + 0.12s_{structure} + 0.10s_{consistency} + 0.08s_{scaleMean}
\]
then clipped to $[0,1]$.

\subsubsection{Calibrated Decision Layer (\fpath{service.py})}
Service calibration transforms raw extractor outputs into stable production decisions:
\begin{enumerate}
  \item computes \texttt{correlation\_margin} and composite \texttt{signal\_score},
  \item applies strict threshold set for non-raw hits,
  \item applies relaxed-but-guarded thresholds if raw extractor already fired,
  \item emits final \texttt{is\_watermarked} and \texttt{signal\_band} in \{strong, weak, negative\}.
\end{enumerate}

Composite signal score in service:
\[
signal = 0.26c_{confidence}+0.22c_{phase}+0.20c_{corr}+0.16c_{carrier}+0.10c_{structure}+0.06c_{consistency}
\]

This two-stage design is critical:
\begin{enumerate}
  \item stage 1 maximizes sensitivity to watermark structure,
  \item stage 2 controls false positives through explicit calibration thresholds.
\end{enumerate}

\subsection{ResNet + ViT + Forensics Deep Dive (\fpath{backend/combine_model.py})}
\subsubsection{Score Construction}
Core score symbols:
\begin{enumerate}
  \item $p_r$: ResNet AI probability,
  \item $p_v$: ViT AI probability (optional),
  \item $p_{legacy}$: legacy model fusion score,
  \item $p_c$: calibrated custom forensic score,
  \item $p_f$: forensics-only score,
  \item $p_{final}$: final image AI probability before threshold compare.
\end{enumerate}

Legacy fusion:
\[
p_{blend}=\frac{p_r+p_v}{2}
\]
with optional meta-learner correction (or blended fallback when version drift/high disagreement is detected).

Custom forensic weighted score from \fpath{backend/custom_forensics.py}:
\[
p_{weighted}=0.50p_r+0.20p_{freq}+0.20p_{anatomy}+0.10p_{entropy}
\]

\subsubsection{Forensic Support Classes}
Each forensic module is categorized as:
\begin{enumerate}
  \item supporting if score $\ge 0.60$ or detector flag is positive,
  \item negative if score $\le 0.15$,
  \item neutral otherwise.
\end{enumerate}

Support/negative counts directly influence threshold selection and disagreement guard behavior.

\subsubsection{Disagreement and Consensus Guards}
After baseline fusion, predictor applies:
\begin{enumerate}
  \item high-confidence ViT disagreement guard (raises AI probability only with corroborating forensic context),
  \item real disagreement guard (reduces false AI outcomes in weak-support contexts),
  \item score-consensus refinement based on ResNet/ViT/legacy agreement margin.
\end{enumerate}

Consensus score when ViT exists:
\[
p_{consensus}=0.50p_v+0.30p_{legacy}+0.20p_r
\]

\subsubsection{Dynamic Threshold Resolver}
Threshold $t$ is not fixed. It depends on:
\begin{enumerate}
  \item supporting module count,
  \item negative module count,
  \item forensics-only score,
  \item ViT/ResNet disagreement,
  \item anatomy availability,
  \item consensus margin.
\end{enumerate}

Important safety behavior:
\begin{enumerate}
  \item low-corroboration guard forces $t \ge 0.86$ when anatomy is unavailable and forensic corroboration is weak/negative,
  \item threshold lowering is blocked in that guarded state.
\end{enumerate}

Final decision rule:
\[
\text{AI if } p_{final} \ge t,\; \text{else Real}
\]

Confidence is threshold-relative margin, not raw AI probability:
\[
conf=0.5+0.5\,\mathrm{clip}(m,0,1)
\]

\subsection{Video Pipeline Deep Dive (\fpath{backend/video_detect_standalone.py})}
\subsubsection{Branch Selection}
Video analysis always attempts dedicated checkpoint inference first.
If checkpoint dependencies or files are missing/failing, execution falls back to frame-level image predictor scoring.

\subsubsection{Dedicated Checkpoint Branch}
Requires both:
\begin{enumerate}
  \item \fpath{backend/checkpoints/efficientnet.onnx},
  \item \fpath{backend/checkpoints/model.pth}.
\end{enumerate}

Behavior:
\begin{enumerate}
  \item sample 3 frames,
  \item run converted ONNX/PyTorch model,
  \item aggregate mean fake probability,
  \item classify \texttt{FAKE} if $p \ge 0.60$.
\end{enumerate}

\subsubsection{Frame Fallback Branch}
Fallback behavior:
\begin{enumerate}
  \item sample 6 frames,
  \item call image predictor probability per frame,
  \item compute mean and peak probabilities,
  \item aggregate with weighted blend,
  \item classify \texttt{FAKE} if final score $\ge 0.65$.
\end{enumerate}

Aggregation:
\[
p_{video}=\mathrm{clip}(0.7\,p_{mean}+0.3\,p_{peak},0,1)
\]

Frame-level flag threshold for explainability metrics is $0.68$.
